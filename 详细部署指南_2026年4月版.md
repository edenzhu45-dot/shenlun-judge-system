## 4. Supabase数据库配置

### 4.1 注册Supabase账户
1. 访问 https://supabase.com
2. 点击右上角 **"Start your project"**
3. 选择登录方式：
   - 推荐使用 **"Continue with GitHub"**
4. 授权GitHub访问

### 4.2 创建新项目
1. 登录后点击 **"New project"** 按钮
2. 填写项目信息：
   - **Name**: `shenlun-judge-db`
   - **Database Password**: 设置一个强密码（记住它！）
   - **Region**: 选择离您最近的区域（如 `Singapore (Southeast Asia)`）
   - **Pricing Plan**: 选择 **Free**
3. 点击 **"Create new project"**
   - 创建过程需要1-2分钟

### 4.3 获取数据库连接信息
项目创建完成后：
1. 点击左侧菜单 **"Settings"**
2. 选择 **"Database"**
3. 找到 **"Connection string"** 部分
4. 点击 **"URI"** 旁边的复制按钮

您会得到类似这样的连接字符串：
```
postgresql://postgres:[YOUR-PASSWORD]@db.[YOUR-PROJECT-REF].supabase.co:5432/postgres
```

### 4.4 创建数据库表
1. 点击左侧菜单 **"SQL Editor"**
2. 点击 **"New query"**
3. 复制并执行以下SQL语句：

```sql
-- 创建激活码表
CREATE TABLE IF NOT EXISTS activation_codes (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    code VARCHAR(50) UNIQUE NOT NULL,
    user_id VARCHAR(100),
    is_used BOOLEAN DEFAULT FALSE,
    is_valid BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    used_at TIMESTAMP WITH TIME ZONE,
    expires_at TIMESTAMP WITH TIME ZONE,
    market VARCHAR(50)
);

-- 创建用户结果表
CREATE TABLE IF NOT EXISTS user_results (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    user_id VARCHAR(100) NOT NULL,
    activation_code VARCHAR(50) NOT NULL,
    file_name VARCHAR(255),
    file_size INTEGER,
    status VARCHAR(50) DEFAULT 'pending',
    score INTEGER,
    feedback TEXT,
    r2_url TEXT,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    completed_at TIMESTAMP WITH TIME ZONE,
    processing_time INTEGER
);

-- 创建索引
CREATE INDEX IF NOT EXISTS idx_activation_codes_code ON activation_codes(code);
CREATE INDEX IF NOT EXISTS idx_activation_codes_user_id ON activation_codes(user_id);
CREATE INDEX IF NOT EXISTS idx_user_results_user_id ON user_results(user_id);
CREATE INDEX IF NOT EXISTS idx_user_results_activation_code ON user_results(activation_code);
CREATE INDEX IF NOT EXISTS idx_user_results_status ON user_results(status);

-- 插入测试激活码
INSERT INTO activation_codes (code, user_id, is_used, is_valid, expires_at, market)
VALUES 
    ('TEST123456', 'test_user', false, true, NOW() + INTERVAL '30 days', 'test'),
    ('TEST789012', 'test_user2', false, true, NOW() + INTERVAL '30 days', 'test')
ON CONFLICT (code) DO NOTHING;
```

4. 点击 **"Run"** 按钮执行

### 4.5 获取API密钥
1. 点击左侧菜单 **"Settings"**
2. 选择 **"API"**
3. 找到 **"Project API keys"** 部分
4. 复制以下信息：
   - **URL**: `https://[YOUR-PROJECT-REF].supabase.co`
   - **anon public**: 公钥（以 `eyJ` 开头）
   - **service_role secret**: 私钥（以 `eyJ` 开头，**不要泄露！**）

## 8. 代码上传到GitHub（不使用GitHub Desktop）

### 8.1 方法一：使用Git命令行（推荐，最专业）
如果您已经安装了Git，这是最标准的方法：

1. **打开命令提示符**：
   - 按 `Win + R` 键
   - 输入 `cmd` 回车

2. **进入项目目录**：
   ```bash
   cd E:\work2\全新网页版
   ```

3. **初始化Git仓库**：
   ```bash
   git init
   ```

4. **添加所有文件**：
   ```bash
   git add .
   ```

5. **提交文件**：
   ```bash
   git commit -m "初始提交：申论智能判卷系统完整代码"
   ```

6. **连接到GitHub远程仓库**：
   ```bash
   git remote add origin https://github.com/您的用户名/shenlun-judge-system.git
   ```

7. **推送代码**：
   ```bash
   git push -u origin main
   ```

8. **如果遇到错误**（仓库已有README文件）：
   ```bash
   git pull origin main --allow-unrelated-histories
   git push -u origin main
   ```

### 8.2 方法二：使用GitHub网页上传（文件太多不适用）
**注意**：GitHub网页最多支持100个文件同时上传，我们的项目文件超过100个，所以**不能使用这个方法**。

### 8.3 方法三：使用压缩包上传（备用方案）

1. **压缩项目文件**：
   - 在 `E:\work2\全新网页版` 文件夹上右键
   - 选择 `7-Zip` → `添加到压缩包...`

   - 点击 `确定`

2. **上传到GitHub**：
   - 访问您的GitHub仓库页面
   - 点击 `Add file` → `Upload files`
   - 拖拽zip文件到上传区域
   - 点击 `Commit changes`

3. **在Render中解压**：
   需要在Render的构建命令中添加解压步骤（后面会说明）

## 13. 故障排除

### 13.1 常见问题及解决方法

#### 问题1: 部署失败，显示"Build failed"
**可能原因**: 依赖安装失败
**解决方法**:
1. 检查 `requirements.txt` 文件格式
2. 查看构建日志中的具体错误
3. 常见问题：
   - 包版本冲突：尝试固定版本号
   - 内存不足：Free计划只有512MB内存

#### 问题2: 应用启动后立即崩溃
**可能原因**: 环境变量缺失或错误
**解决方法**:
1. 检查所有必需的环境变量是否已设置
2. 检查环境变量值是否正确
3. 查看应用日志：
   - 在Render控制台，点击 **"Logs"** 标签
   - 查看错误信息

#### 问题3: 无法连接到数据库
**可能原因**: Supabase连接配置错误
**解决方法**:
1. 检查 `SUPABASE_URL` 格式
2. 检查 `SUPABASE_KEY` 是否正确
3. 在Supabase控制台检查数据库状态

#### 问题4: 文件上传失败

#### 问题5: AI判卷失败

**解决方法**:
1. 检查 `DEEPSEEK_API_KEY` 是否正确
2. 登录DeepSeek平台检查额度

## 14. 支持500名用户的完整配置方案

### 14.1 500名用户的需求分析
假设500名用户的使用场景：
- **每月活跃用户**: 500人
- **每人每月判卷次数**: 5次
- **每月总判卷次数**: 2500次
- **每次判卷PDF大小**: 平均5MB
- **每月存储需求**: 2500次 × 5MB = 12.5GB
- **每月AI调用**: 2500次

### 14.2 必需升级的服务配置

#### 1. Render升级（必须）
- **计划**: **Starter计划** ($7/月)
- **原因**: 免费计划会休眠，500用户无法接受30秒启动等待
- **配置**:
  - 内存: 512MB → 1GB（升级到Pro计划）
  - CPU: 共享 → 专用
  - 无休眠机制
  - 自定义域名支持

#### 2. Supabase升级（必须）
- **计划**: **Pro计划** ($25/月)
- **原因**: 免费计划只有500MB存储，不够用
- **配置**:
  - 数据库存储: 500MB → 8GB
  - 带宽: 大幅增加
  - 身份验证: 每月50,000用户（足够）
  - 备份: 自动每日备份

#### 3. Cloudflare R2升级（必须）
- **计划**: **付费计划** (按使用量付费)
- **原因**: 免费计划只有10GB存储，500用户需要更多
- **配置**:
  - 存储: 10GB → 至少50GB
  - 操作: 每月1,000,000次 → 无限制（按量付费）
  - 费用估算: $5-10/月（12.5GB存储 + 2500次操作）

#### 4. DeepSeek API充值（必须）
- **计划**: **按量付费**
- **原因**: 免费额度不够2500次调用
- **费用估算**:
  - 每次判卷约1000 tokens
  - 2500次 × 1000 tokens = 2,500,000 tokens
  - DeepSeek价格: 约$0.14/百万tokens
  - 每月费用: 2.5 × $0.14 = $0.35

### 14.3 500用户完整配置清单

| 服务 | 计划 | 月费用 | 配置说明 |
|------|------|--------|----------|
| Render | Pro计划 | $25 | 1GB内存，无休眠，专用CPU |
| Supabase | Pro计划 | $25 | 8GB存储，50,000用户/月 |
| Cloudflare R2 | 按量付费 | $5-10 | 50GB存储，无限制操作 |
| DeepSeek API | 按量付费 | $0.35 | 250万tokens/月 |
| **总计** | **-** | **$55-60** | **支持500名用户** |

#### 步骤1: 升级Render到Pro计划
1. 登录Render控制台
2. 点击您的Web Service
3. 点击右上角 **"Settings"**
