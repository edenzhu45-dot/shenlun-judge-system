# 申论判卷系统 - 最终操作指南

## 第一步：解决所有问题（按顺序）

### 1. GitHub仓库地址问题
**正确地址格式**：`https://github.com/您的用户名/仓库名.git`
**您的用户名**：登录GitHub后，右上角显示的名字
**创建仓库**：https://github.com/new
**仓库名建议**：`shenlun-judge-system`

### 2. Supabase配置问题
**操作步骤**：
1. 访问：https://supabase.com
2. 点击 "Start your project"
3. 创建项目，获取：
   - **URL**：`https://xxxxxx.supabase.co`
   - **anon public key**：以 `eyJ` 开头
   - **service_role secret**：以 `eyJ` 开头（私钥）

**详细步骤**：查看 `supabase_simple_guide.md`

### 3. Redis配置问题
**最简单方案**：使用内存队列（不需要外部Redis）
**操作**：什么都不用做，代码已支持内存队列

**如果需要Redis**：查看 `redis_simple_guide.md`

### 4. Git命令错误问题
**正确命令**：`git init`（不是 `git inti`）
**一键解决方案**：双击运行 `upload.bat`

### 5. 文件上传超过25MB问题
**核心原因**：`venv` 文件夹太大
**解决方案**：运行 `upload.bat` 自动清理

## 第二步：一键上传操作

### 方法A：使用脚本（最简单）
1. 双击 `upload.bat`
2. 按照提示操作
3. 输入GitHub仓库地址

### 方法B：手动操作
```bash
# 1. 打开命令提示符
Win + R，输入 cmd，回车

# 2. 进入项目目录
cd E:\work2\全新网页版

# 3. 删除大文件
rmdir /s /q venv
rmdir /s /q __pycache__

# 4. 初始化Git
git init

# 5. 添加文件
git add .

# 6. 提交
git commit -m "申论判卷系统"

# 7. 连接到GitHub
git remote add origin https://github.com/您的用户名/shenlun-judge-system.git

# 8. 推送
git push -u origin main
```

## 第三步：验证上传成功

1. **访问GitHub**：https://github.com/您的用户名/shenlun-judge-system
2. **检查文件**：应该看到这些文件夹：
   - `backend/`
   - `frontend/`
   - 各种配置文件

3. **文件数量**：大约30-40个文件
4. **文件大小**：小于5MB（清理后）

## 第四步：部署到Render

### 准备工作
需要的信息：
1. **GitHub仓库地址**：`https://github.com/您的用户名/shenlun-judge-system`
2. **Supabase信息**：URL和API密钥
3. **DeepSeek API密钥**：从 https://platform.deepseek.com 获取

### 部署步骤
1. 访问：https://render.com
2. 点击 "New +" → "Web Service"
3. 连接GitHub仓库
4. 填写配置：
   - **Name**：`shenlun-judge`
   - **Environment**：Python 3
   - **Build Command**：`pip install -r requirements.txt`
   - **Start Command**：`python main.py`
5. 添加环境变量（从 `.env.example` 复制）
6. 点击 "Create Web Service"

## 故障排除

### 问题1：GitHub上传失败
**解决**：
1. 确认网络连接
2. 确认GitHub账号密码正确
3. 使用HTTPS地址，不是SSH

### 问题2：Render部署失败
**解决**：
1. 检查 `requirements.txt` 格式
2. 检查环境变量是否正确
3. 查看Render的日志

### 问题3：系统无法运行
**解决**：
1. 检查所有API密钥是否正确
2. 检查Supabase表是否创建
3. 检查DeepSeek API是否可用
