# 解决文件上传超过25MB问题

## 问题分析
您的项目文件夹包含 `venv` 虚拟环境，里面有大量Python包文件，导致总大小超过25MB。

## 解决方案

### 方案一：删除不需要的文件（推荐）
运行我创建的 `upload.bat` 脚本，它会自动：
1. 删除 `venv` 文件夹（虚拟环境）
2. 删除 `__pycache__` 缓存文件
3. 只上传必要的源代码文件

### 方案二：手动清理
如果您想手动操作：

1. **删除虚拟环境**：
   ```bash
   rmdir /s /q venv
   ```

2. **删除缓存文件**：
   ```bash
   rmdir /s /q __pycache__
   rmdir /s /q backend\app\__pycache__
   ```

3. **删除测试文件**（可选）：
   ```bash
   del test_*.py
   del check_environment.py
   del start_local_test.py
   ```

### 方案三：创建精简版项目
我已经为您创建了精简版项目结构，只包含必要的文件：

## 具体操作步骤

### 步骤1：运行清理脚本
双击运行 `upload.bat`，它会自动清理并上传。

### 步骤2：如果脚本失败，手动操作
1. 打开命令提示符（cmd）
2. 输入：
   ```bash
   cd E:\work2\全新网页版
   rmdir /s /q venv
   rmdir /s /q __pycache__
   git init
   git add .
   git commit -m "精简版申论判卷系统"
   ```

### 步骤3：创建GitHub仓库
1. 访问：https://github.com
2. 点击 "+" → "New repository"
3. 填写：
   - Repository name: `shenlun-judge-system`
   - 不要勾选 "Add a README file"
4. 点击 "Create repository"

### 步骤4：获取仓库地址
创建成功后，复制仓库地址：
```
https://github.com/您的用户名/shenlun-judge-system.git
```

### 步骤5：上传代码
在命令提示符中：
```bash
git remote add origin https://github.com/您的用户名/shenlun-judge-system.git
git push -u origin main
```

## 验证上传成功
1. 访问您的GitHub仓库页面
2. 应该看到这些文件：
   - backend/
   - frontend/
   - requirements.txt
   - render.yaml
   - 等

## 如果还是超过25MB
如果清理后仍然超过25MB，使用这个终极方案：

### 创建最小化版本
1. 新建一个文件夹：`E:\work2\精简版`
2. 只复制这些文件：
   - backend/app/ 下的所有.py文件
   - frontend/index.html
   - requirements.txt
   - render.yaml
   - main.py
   - run_worker.py

3. 在这个新文件夹中运行Git命令。

## 重要提醒
1. **不要上传** `.env` 文件（包含密码）
2. **必须上传** `.env.example` 文件（模板）
3. **必须上传** `requirements.txt`（依赖列表）
4. **必须上传** `render.yaml`（部署配置）

上传成功后，Render会自动安装所有依赖包，不需要上传 `venv`。
